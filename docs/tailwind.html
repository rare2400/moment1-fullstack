Tailwind CSS

Länk till Tailwind CSS webbplats

Till en början var Tailwind ovant och det kändes rätt tungt att skriva in många klasser direkt i HTML-koden, samt ha koll på alla klasser som fanns tillgängliga. Sedan fick jag lära mig att tillägget Tailwind CSS intellisense fanns att installera till Visual Studio Code, som hjälper till att hitta klasser och ge alternativ på till exempel färger och storlekar. Detta underlättade mycket i utvecklingen, men det har ändå tagit lite tid att vänja sig vid denna typ av metod. Upplevelsen efter att ha arbetat med Tailwind CSS har varit blandad. Stundvis känns det väldigt smidigt och enkelt men när det exempelvis kom till stylingen av tabellen var det mycket upprepning av klasser. När man vill göra mindre förändringar kräver det att man ändrar i varje cell, vilket känts lite drygt även att det går att skriva på flera rader samtidigt. Samtidigt som det är skönt att kunna välja utseendet själv och anpassa efter sina egna önskemål.

Tailwind CSS är ett utilitybaserat ramverk som bygger på att man applicerar klasser för att styla element direkt i HTML-koden. Stylingen väljer man alltså själv tillsammans med de utilityklasserna som finns i ramverket. Det finns alltså inga färdiga komponenter utan man skapar all styling själv, precis som man gör när man arbetar i en separat CSS-fil. I detta arbete har Tailwinds CLI-verktyg använts för att skapa en CSS-fil innehållande de klasser som behövs för att styla webbplatsen. Vill man addera till exempel egna färger att använda kan man skapa sina egna klasser i den CSS-fil som Tailwind CSS har importerats i.

Prestandan för webbplatser byggda med Tailwind CSS kan se olika ut beroende på vilken metod man väljer att använda. Hämtar man CSS-klasserna genom CDN i head-sektionen är det många rader kod som läses in och det är endast rekomenderat för utveckling, inte för en slutprodukt. I detta arbete har Tailwind CLI använts som endast skapar en CSS fil anpassad för den koden som används i projektet. Det finns även möjlighet att minifiera sin CSS när projektet byggs, vilket ger en bättre prestanda. Därmed är Tailwind CSS ett ramverk som är väl anpassat för att upprätthålla en bra prestanda för webbplatser som byggs med hjälpa av det.

Tailwind har mycket dokumentation på sin hemsida, med tydliga exempel och förklaringar för hur det fungerar. Det är även ett stort community som använder sig av ramverket och det finns forum på Github och en discord server för att få hjälp av andra användare. Det går även att hitta cheat sheets publicerade, där man kan får stöd under utvecklingen, för vilka klasser som finns i ramverket. Ramverket är väl etablerat och används av många utvecklare, det uppdateras även regelbundet vilket ger en långsiktig trygghet i att använda det.
