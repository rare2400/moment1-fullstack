Publicering

Jag har valt att använda Github Pages för att publicera min webbplats, som till en början är gratis att använda. Tidigare har jag, förutom miun.studenter.se som är ftp-baserat, använt mig av Netlify för att publicera webbapplikationer. Till denna uppgiften har jag valt att testa Github Pages som publiceringsplattform. Plattformen stöder statiska webbplatser med programmeringssptåk som HTML, CSS och JavaScript, därmed kan den inte hantera backendapplikationer med exempelvis Node.js. Däremot kan frontend-applikationer med API-anrop till publicerade webbtjänster läggas upp med Github Pages. Som tidigare nämnt så är plattformen gratis när det kommer till mer lättviktiga projekt, skulle man däremot behöva mer bandbredd eller vilja ha en egen domän finns det möjlighet till att betala för detta.

Att Github Pages är gratis är en stor fördel då det är mycket tillgängligt för mindre webbapplikationer, eller som i detta fall en mindre skoluppgift. Eftersom Github används som versionshanterare så går det snabbt och enkelt att uppdatera sin publicerade sida, vilket gör det CI/CD (Continiuous Integration & Continiuous Delivery/Deployment). Fördelen med detta är att flera utvecklare kan arbeta i samma projekt, i olika branches för att sedan mergea den till main, som då direkt uppdaterar sidan automatiskt. Man behöver alltså inte tänka på att ladda upp hela projektet till Github, utan det ligger redan där från början eftersom det även används som versionshanterare.

Plattformen Github Pages stöder inte publicering av exempelvis webbtjänster vilket kan vara en nackdel i de situationer som man arbetar med fullstacks-utveckling. Har man skapat i en backend-applikation behöver den alltså publiceras på en annan plattform, i många fall då med en annan domän. Detta kan kännas som mer jobb för utvecklaren och vid utvecklingen av webbtjänsten behöver man tänka på detaljer som CORS, för att domänerna inte ska krocka vid hämtning av data ur API:et. Min upplevelse av att publicera en webbplats med Github Pages var att det var enkelt och gick väldigt snabbt att gör. Det är även skönt att publicering och versionshantering sker på samma plats, vilket gör publicering enklare.
